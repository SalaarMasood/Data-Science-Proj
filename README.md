# Using Machine Learning to Unlock Insights from Pakistan's Largest E-Commerce Dataset

A data science project for **CS334 (Data Science)** at **LUMS** that applies exploratory data analysis and machine learning to Pakistan's largest publicly available e-commerce dataset.

---

## 📦 Dataset

**Pakistan's Largest E-Commerce Dataset** — sourced from [Kaggle](https://www.kaggle.com/datasets/zusmani/pakistans-largest-ecommerce-dataset).

- **~1 million** raw transaction records (July 2016 – August 2018)
- **21 features** after cleaning: order status, price, quantity, grand total, discount, payment method, product category, customer ID, timestamps, and more.

> **Note:** The preprocessed dataset (`preprocessed_dataset.csv`) is excluded from this repository via `.gitignore` due to its large size (~91 MB). Download the raw dataset from Kaggle and run `DataCleaning.ipynb` to reproduce it.

## 🔍 Phase 1 – Data Cleaning & EDA (`DataCleaning.ipynb`)

Key preprocessing steps:
- Downloaded dataset via `kagglehub`
- Dropped trailing empty columns (`Unnamed: 21–25`)
- Removed fully-null rows
- Standardised column names (lowercase, snake_case)
- Assigned appropriate data types (`int`, `float64`, `object`)
- Exported cleaned data to `preprocessed_dataset.csv`

Exploratory analysis covered sales trends, category distributions, payment method breakdowns, and seasonal patterns.

---

## 🤖 Phase 2 – Machine Learning Models (`models.ipynb`)

Three research questions are addressed:

### RQ1 — Order Completion Prediction (Classification)

> *Can we predict whether an order will be completed or cancelled?*

Two models trained side-by-side:

| Model | Approach | AUC |
|---|---|---|
| **Random Forest** | `TimeSeriesSplit(n_splits=5)`, 200 estimators, balanced class weights | **0.707** |
| **Neural Network (MLP)** | Dense[128→64] + BatchNorm + LeakyReLU + Dropout(0.3), early stopping on val AUC | **0.731** |

**Top predictive features:** `grand_total`, `price`, `discount_amount`, `payment_method_cash_on_delivery`

---

### RQ2 — Customer Segmentation (Clustering)

> *Can we group customers into meaningful clusters?*

- **Algorithm:** K-Means clustering
- **Elbow method + Silhouette scores** used to select optimal *k*
- Customers segmented by purchase behaviour, value, and frequency

---

### RQ3 — Sales Forecasting (Regression / Time Series)

> *Can we predict future sales based on historical patterns?*

- **Algorithm:** Linear Regression on aggregated monthly/weekly sales
- Evaluated with RMSE, MAE, and R²

## 🚀 Getting Started

### Prerequisites

```bash
pip install pandas numpy matplotlib seaborn scikit-learn tensorflow kagglehub tqdm
```

### Running the Notebooks

1. **Clone the repository:**
   ```bash
   git clone https://github.com/SalaarMasood/Data-Science-Proj.git
   cd Data-Science-Proj
   ```

2. **Run Phase 1 – Data Cleaning:**
   Open `DataCleaning.ipynb` in Jupyter and run all cells. This downloads the dataset from Kaggle and produces `preprocessed_dataset.csv`.

3. **Run Phase 2 – Models:**
   Open `models.ipynb` in Jupyter and run all cells. Requires `preprocessed_dataset.csv` in the project root.

> **Kaggle API credentials** are required for `kagglehub`. Follow [Kaggle's API setup guide](https://www.kaggle.com/docs/api) to configure them.
